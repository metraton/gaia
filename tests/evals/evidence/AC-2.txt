What landed (closing the AC-2 follow-up)
----------------------------------------
1. catalog.CaseModel now carries `expected_decision` (Optional[str]), validated
   against VALID_DECISIONS = (allow, ask, deny) when present, None elsewhere.
   load_catalog no longer drops the curated oracle key.
2. runner.HookLogReplayBackend replays each case's command through the REAL
   PreToolUse entry point (hooks/pre_tool_use.py) as a SUBPROCESS — never an
   in-process import, so runner.py's "MUST NOT import from hooks/" contract is
   preserved (verified: grep finds no `import hooks` / `from hooks` in
   runner.py). It delegates the subprocess plumbing to the existing
   tools/gaia_simulator/runner.py HookRunner so the replay path stays
   consistent with the gaia_simulator regression detector.
3. Each dispatch runs against an isolated GAIA_DATA_DIR (fresh, empty gaia.db
   per case) so the approval store from one replayed T3 cannot poison the next
   (without it, the first T3 writes a pending grant that a later replay sees as
   prior state and escalates ask->deny). GAIA_PLUGIN_MODE is pinned to `ops` so
   the replay deterministically exercises Gaia's own T3 nonce path regardless
   of the ambient mode the test process inherited.

Oracle mapping (decision = policy, not transport)
-------------------------------------------------
The observed hook outcome is classified by the reason marker the security core
stamps, NOT by exit code (the ops-mode T3 block and a permanent block share
exit 2), making the oracle plugin-mode-independent:
  - [BLOCKED] (blocked_commands.py, never approvable, exit 2)  -> deny
  - [T3] / [T3_BLOCKED] (consent required; ops-mode hard block is still
    approvable via an approval_id; security-mode native ask)   -> ask
  - native permissionDecision allow                            -> allow

Two oracle corrections made during wiring (observed reality wins)
-----------------------------------------------------------------
- SEC-LOCAL-git-commit: message changed from 'wip' to 'chore: wip'. The case
  asserts the TIER decision (git commit is local-only -> allow); a
  non-conventional message tripped the orthogonal commit-message validator and
  turned it into an ask, masking what the case tests.
- SEC-COMP-curl-pipe-sh: expected_decision corrected ask -> deny. composition_
  rules.py classifies network_read | exec_sink (curl ... | sh) as a PERMANENT
  block (RCE rule, exit 2, never approvable), not a T3 escalation — its own
  module docstring is explicit. To preserve "dangerous composition -> consent"
  coverage, a new SEC-COMP-file-to-exec case (cat ./setup.sh | bash) was added,
  which composition_rules.py DOES route to T3 ask (file_read | exec_sink
  escalation). The corpus now pins the actual implementation policy, not the
  security-tiers gloss that lumps all composition into ask.
